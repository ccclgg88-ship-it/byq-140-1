## 1. 项目概述

### 1.1 网站名称

个人简历展示网站

### 1.2 网站定位

面向高职院校「广告艺术设计」专业毕业生的个人简历展示**静态网站**，突出「设计+技术」双维度能力，为求职提供专业的线上展示平台。

### 1.3 核心目标

- 用设计感快速抓住 HR 注意力，明确求职方向。
- 用一句话定义核心价值，突出「设计+技术」差异化优势。
- 清晰展示设计技能与计算机信息能力，证明双专业优势。
- 用具体项目证明「设计+技术」结合能力与实战经验。
- 补充教育背景，表达求职诚意。

### 1.4 约束与范围

- **至少 5 个独立静态 HTML 页面**，含 CSS 样式表、JS 文件、图片文件夹等。
- **纯前端实现**，无后端服务；国际化语言为**中文**。
- 代码尽量简单，能覆盖和实现功能即可。

## 2. 技术栈与工程架构

### 2.1 技术栈

| 类别     | 技术           | 用途说明                                   |
|----------|----------------|--------------------------------------------|
| 结构     | HTML5          | 语义化标签、表单控件、媒体元素             |
| 样式     | CSS3           | Flex/Grid 布局、动画、过渡、响应式设计     |
| 行为     | JavaScript     | DOM 操作、事件监听、表单验证、localStorage |
| 组件库   | Bootstrap      | 响应式栅格、导航栏、卡片、轮播图           |
| 交互组件 | jQuery UI      | 日期选择器、对话框等                       |
| 移动端   | jQuery Mobile  | 触屏优化、移动端适配                       |

### 2.2 工程架构与模块划分

采用「多页静态 + 公共资源」的简单架构，便于维护与扩展。

```
项目根目录/
├── index.html              # 封面页（入口）
├── positioning.html        # 个人定位页
├── skills.html             # 技能与工具页
├── works.html              # 作品与项目页
├── education.html          # 教育与总结页
├── css/
│   ├── common.css          # 公共样式（配色、字体、布局基础）
│   ├── cover.css           # 封面页样式（可选，或合并到 common）
│   └── ...                 # 其他页面专属样式（按需拆分）
├── js/
│   ├── common.js           # 公共逻辑（导航、本地存储、工具函数）
│   └── ...                 # 各页专属交互（如表单验证、轮播）
├── images/                 # 图片资源
│   ├── avatar.jpg          # 个人职业照
│   └── ...                 # 作品图、图标等
└── doc/                    # 文档（本需求说明等）
```

### 2.3 模块职责简述

| 模块         | 职责说明                                           |
|--------------|----------------------------------------------------|
| 公共 CSS     | 统一配色（蓝+橙）、字体（标题黑体、正文微软雅黑）、栅格与留白 |

## 3. 核心需求与页面规格

### 3.1 封面页（index.html）

**核心目标**：用设计感快速抓住 HR 注意力，明确求职方向。

| 需求项       | 内容说明 |
|--------------|----------|
| 视觉元素     | 个人职业照（简洁背景，体现设计审美） |
| 创意标题     | 「广告设计×技术落地 \| 用设计解决问题的实践者」 |
| 关键信息     | 姓名：雷青青；求职岗位：广告设计师/视觉设计师/UI设计师 |
| 身份信息     | 现任团支书，计算机信息学院学生会主席团成员 |
| 核心技能标签 | PS/AI、C4D、Figma、HTML/CSS（图标+文字组合） |
| 联系方式     | 电话、邮箱、作品集链接（如 Behance/个人网站） |

**设计要点**：

- 配色：高饱和度对比色（蓝+橙、黑+白），体现专业感。
- 布局：非对称（如左侧照片+右侧文字），避免呆板。
- 整体风格符合广告设计常用配色，突出专业与创意。

**核心目标**：用一句话定义核心价值，突出「设计+技术」差异化优势。

| 需求项       | 内容说明 |

### 3.3 技能与工具页（skills.html）

**核心目标**：清晰展示「设计技能」与「计算机信息能力」，证明双专业优势。

| 需求项               | 内容说明 |
|----------------------|----------|
| 设计技能-软件        | Photoshop（精通）、Illustrator（精通）、C4D（熟练）、Figma（熟练） |
| 设计技能-能力        | 平面设计、UI 设计、品牌视觉、海报设计（可配案例支撑） |
| 技术能力-工具        | HTML/CSS（基础）、Python（基础，用于设计自动化）、Excel（数据分析） |

**设计要点**：

- 左右分栏：左侧设计技能，右侧技术能力。
- 每个技能配小图标（如 PS 图标、代码图标）增强可读性。
- 星级或进度条表示熟练度，直观展示技能水平。

### 3.4 作品与项目页（works.html）

**核心目标**：用具体项目证明「设计+技术」结合能力，突出实战经验。

| 项目类型           | 项目名称                 | 要点摘要 |
|--------------------|--------------------------|----------|

| 技术赋能设计类     | 交互式产品宣传页设计     | Figma 原型→HTML/CSS 静态页→JavaScript 交互；访问量 1000+，客户反馈交互体验提升 |

**设计要点**：

- 大图+文字组合（如左侧作品图，右侧项目说明）。
- 技术关键词用橙色或高亮区分（如 HTML/CSS 实现、Figma 交付规范）。
- 成果数据化表达（效率提升 50%、转化率 15% 等）。

**核心目标**：补充教育背景，强化「设计+技术」专业基础，表达求职意愿。

| 需求项       | 内容说明 |
|--------------|----------|
| 教育背景     | 院校：高职院校计算机信息学院；专业：广告设计；核心课程：设计类（平面构成、广告创意）+ 技术类（网页设计、Python 基础） |

**设计要点**：

- 课程以简洁列表展示，避免冗长。
- 结尾用大字号加粗突出求职意愿。
- 强调「设计+技术」结合的价值观。

## 4. 设计规范（全站统一）

### 4.1 视觉一致性

| 项目       | 规范说明 |
|------------|----------|
| 配色方案   | 主色：广告设计常用蓝色；辅助色：橙色 |
| 字体       | 标题：黑体；正文：微软雅黑 |
| 技术标签   | 图标+高亮突出技术关键词（如 HTML/CSS、Python） |

### 4.2 数据化表达

- 成果均用数字支撑（如效率提升 50%、转化率提升 15%、印刷 500 份、访问量 1000+）。
- 避免空泛描述，用具体数据体现价值。

### 4.3 技术标识

- 各页面突出技术关键词，便于 HR 快速识别技术能力。
- 与纯艺术设计毕业生区分，体现复合型人才优势。

### 4.4 排版与结构

- 标题层级：标题 1（#）→ 标题 2（##）→ 标题 3（###），逻辑清晰。
- 段落间距：标题与正文空一行，正文段落 1.5 倍行距。
- 列表：技术栈、内容结构用无序列表；步骤类用有序列表。
- 重点：技术词加粗或高亮，数据用数字强调。

## 5. 非功能需求与实现原则

### 5.1 实现原则

- **简单优先**：代码以覆盖功能、易于维护为主，不做过度抽象。
- **中文呈现**：全部面向中文用户，文案与界面均为中文，无需多语言框架。
- **静态可交付**：仅前端静态资源，可直接用浏览器打开或通过任意静态服务器部署。

### 5.2 兼容与体验

- 使用 Bootstrap 栅格与媒体查询实现响应式，兼顾 PC 与移动端。
- 使用 jQuery Mobile 优化触屏体验（如点击区域、滑动）。
- 若有表单（如联系方式、留言），需前端校验与用户提示（如 Toast）。

### 5.3 可扩展性

- 公共样式与脚本集中放在 `css/common.css`、`js/common.js`，新页面复用即可。
- 图片、作品案例后续可替换为真实素材，不改变页面结构。
