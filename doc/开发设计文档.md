## 1. 设计目标与原则

### 1.1 设计目标

- **完全覆盖需求**：实现需求说明文档中的 5 个页面及全部内容要点、设计规范。
- **工程清晰**：采用「多页静态 + 公共资源」架构，目录与模块职责明确。
- **实现简单**：代码以实现功能、易维护为主，不做过度抽象与复杂封装。

### 1.2 设计原则

- 纯前端静态站点，无后端与数据库；全站中文。

## 4. 全局设计规范（设计令牌）

### 4.1 配色（CSS 变量建议）

在 `common.css` 中定义，全站统一引用：

| 变量名（示例）   | 用途     | 建议值说明           |
|------------------|----------|----------------------|
| `--color-primary`| 主色     | 广告蓝 #1a73e8 或类似 |
| `--color-accent` | 辅助色   | 橙色 #f57c00 或类似   |
| `--color-dark`   | 深色文字 | #1a1a1a / 黑         |
| `--color-light`  | 浅底/白  | #ffffff / #f5f5f5    |
| `--color-tech`   | 技术高亮 | 与辅助色一致或 #f57c00 |

### 4.2 字体

- **标题**：`font-family: "黑体", "SimHei", sans-serif;`
- **正文**：`font-family: "微软雅黑", "Microsoft YaHei", sans-serif;`
- **行高**：正文约 1.5；标题与正文之间留白统一（如 margin-bottom）。

### 4.3 间距与圆角

- 使用统一间距尺度（如 8px 基准：8、16、24、32px）。
- 卡片与按钮使用圆角（如 `border-radius: 8px`）和轻微阴影（`box-shadow`）区分层级。
- 段落间距约 1.5em 或 24px，避免拥挤。

### 4.4 响应式断点（与 Bootstrap 一致）

- 沿用 Bootstrap 5 断点：`sm` 576px、`md` 768px、`lg` 992px、`xl` 1200px。
- 移动端：单栏、导航可折叠或简化为图标；触屏区域足够大（jQuery Mobile 可增强）。

### 4.5 技术关键词展示

- 技术词（如 HTML/CSS、Python、Figma）使用类名如 `.tech-highlight`，背景或文字色使用 `--color-accent` 或橙色高亮。
- 可配合小图标（放在 `images/icons/` 或使用 Unicode/简单 SVG）增强识别。

## 6. 公共模块设计

### 6.1 公共样式（css/common.css）

**建议结构**（顺序可调）：

1. **CSS 变量**：配色、字体、间距、圆角、阴影等。
2. **重置/归一**：可选简单 normalize 或仅统一 box-sizing、body 字体与行高。
3. **布局**：容器最大宽度、居中、通用 padding。
4. **导航**：`.navbar` 或与 Bootstrap 导航结合，高亮当前页（通过 body 的 class 或 data 属性标识当前页，用 `.nav-link.active` 等）。
5. **标题与正文**：h1～h3 黑体，正文微软雅黑，段落间距。
6. **通用组件**：按钮基础样式、卡片基础、`.tech-highlight` 高亮样式。
7. **页脚**：统一版权或「返回顶部」等（若需要）。
8. **响应式**：导航折叠、栅格断点下的间距调整。

### 6.2 公共脚本（js/common.js）

**建议功能**（简单实现即可）：

1. **当前页导航高亮**：根据当前页面 URL 或 body 的 data-page 属性，为对应 nav 链接添加 active 类。
2. **localStorage**：若需求中有「记住访问时间/次数」等，可读写 localStorage 并在某页展示；否则可仅预留工具函数（如 `get/setStorage`）。
3. **Toast 提示**：封装简单函数 `showToast(message)`，用 Bootstrap Toast 或自定义 div 显示 2～3 秒后消失；用于表单校验失败或成功提示。
4. **表单基础校验**：对页面内表单（若有）做非空、邮箱格式等校验，不通过时 showToast 并 return false。

不要求复杂架构，函数式即可。

### 6.3 导航与页间一致

- 每个 HTML 页面包含相同的导航结构（可复制同一段 HTML 或后续用模板思想手写多份），链接到 5 个页面。
- 当前页链接可加 `class="active"` 或 `aria-current="page"`，由 common.js 根据当前路径设置。

## 8. jQuery UI / jQuery Mobile 使用说明

### 8.1 jQuery UI

- **使用场景**：需求中提到「日期选择器、对话框」。若教育页或联系表单有「期望到岗日期」，可加一个 input，用 jQuery UI Datepicker 绑定；若有「查看详情」弹窗，可用 jQuery UI Dialog。按需引入对应组件与主题 CSS，避免全量加载。

### 8.2 jQuery Mobile

- **使用场景**：移动端触屏优化。可引入 jQuery Mobile 的 CSS 与 JS，对导航、按钮、列表等使用 `data-role` 等属性增强触控体验；若仅做简单适配，也可主要依赖 Bootstrap 响应式 + 大点击区域，jQuery Mobile 做少量增强。

实现时以「简单可用」为度，不必堆砌所有组件。
